DEPLOY ON HOSTINGER (domain is already at Hostinger — no nameserver step!)
  1. hPanel -> Websites -> Add website -> Empty website -> select
     middlenamesgenerator.com
  2. Files -> File Manager -> public_html -> delete default files
  3. Upload the ZIP -> right-click -> Extract -> move files so index.html sits
     directly at public_html/index.html (not inside a subfolder)
  4. Security -> SSL -> install free SSL -> enable Force HTTPS
  5. Test on your phone: generate names, tap a 🔊 button, click every nav link
  6. Set up the namesthatflow.com 301 using htaccess-for-namesthatflow.txt

GOOGLE SEARCH CONSOLE (same day)
  Add property middlenamesgenerator.com (domain type, DNS TXT verification in
  hPanel -> Domains -> DNS) -> submit sitemap.xml -> request indexing on the
  homepage and both middle-names pages.

ADSENSE (~2 weeks after launch)
  Compliance pages are in place. On approval, replace the .ad-slot divs with
  your ad code, and enable Google's certified GDPR consent message in
  AdSense -> Privacy & messaging (required for UK/EEA).

GROWTH ENGINE — one hub page per name, 2-4 per week
  Copy middle-names-olivia.html (girls) or -noah.html (boys) and update:
  title/meta/canonical/schema/H1, the rhythm intro (depends how the name ENDS),
  pill lists, the "[Name] name meaning and origin" section, FAQs, and the
  firstName input value. Add to sitemap.xml + footer links.
  Priority queue (validated volume): Amelia, Isla, Ava, Freya, Lily, Ivy,
  Florence, Willow, Elodie, Maeve / Muhammad, Oliver, George, Leo, Arthur,
  Luca, Theodore, Oscar, Elias, Arlo.
  Each page targets: "middle names for X" (primary) + "X name meaning"
  (secondary) + "names that go with X" (tertiary).

TARGET: 50 pages live + Search Console impressions before touching any other
domain or project. That is the whole strategy.
